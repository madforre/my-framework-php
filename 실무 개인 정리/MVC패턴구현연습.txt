라우터 클래스

생성자

	인터페이스 의존성 참조 유지

__call 메소드

	
	건초더미인 배열 HttpMethods 에서 
	라우터 인스턴스에서 호출한 메소드가 있는지 확인
	호출한 메서드가 프로퍼티에 있으면 메소드를 동적으로 생성
	없으면 사용할 수 없는 메소드임을 화면에 랜더링

	첫번째 인수 메소드 이름 ('get')
	두번째는 인수의 배열이다.

formatRoute 메소드

	라우터의 슬래쉬를 제거

resolve 메소드

	Request의 동적 프로퍼티인 requestMethod의 값을 소문자로 변환 후
	해당 HTTP 요청에 해당하는 오버로딩된 메소드를 



클래스의 흐름

1. 웹서버에 접속하면 nginx 서버 설정파일에서 index.php로 이동시킨다.
2. index.php에서 autoload.php 를 불러온다.
3. autoload.php에서 필요한 모듈을 불러온다.(라우터)
4. 리퀘스트 객체를 매개변수로 가지는 라우터 객체를 생성한다.
5. 라우터 객체가 생성될 때 전달받은 매개변수인 리퀘스트 객체를 
   라우터 객체의 프로퍼티에 저장한다.
6. HTTP 요청에 대한 정보들이 가공되어있는 request객체에서 requestUri 를 이용한다.
   requestUri 를 저장한다.
   요청에 대한 정보들이 있는 상태인가?
   웹서버의 다큐먼트 루트를 통해서 들어온 것이지
   서버에서 location설정으로 들어온게 아니여서 실행이 안되는게 아닌가?
