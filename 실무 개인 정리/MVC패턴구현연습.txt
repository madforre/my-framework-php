라우터 모듈 만들고 사용한다.

라우터 클래스

생성자

	인터페이스 의존성 참조 유지

__call 메소드

	
	건초더미인 배열 HttpMethods 에서 
	라우터 인스턴스에서 호출한 메소드가 있는지 확인
	호출한 메서드가 프로퍼티에 있으면 메소드를 동적으로 생성
	없으면 사용할 수 없는 메소드임을 화면에 랜더링

	첫번째 인수 메소드 이름 ('get')
	두번째는 인수의 배열이다.

formatRoute 메소드

	라우터의 슬래쉬를 제거

resolve 메소드

	Request의 동적 프로퍼티인 requestMethod의 값을 소문자로 변환 후
	해당 HTTP 요청에 해당하는 오버로딩된 메소드를
