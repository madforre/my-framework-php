내가 만든 PHP 프레임워크의 약점

- 우분투 방화벽이 설정되있지 않음.

- HTTPS 인증

- URL 필터 (유효성 검사) 가 안된다.

    어떤 면에서? .php 찍으면 내가 만든 404 페이지가 안 뜬다!
    autoload.php 라우터 사용자 설정 확인 전에 구현하다 말았다.
    동적으로 서버변수들을 부트스트래핑해서 원할 때 사용 하고 싶은데
    프로퍼티가 비공개라서 접근이 안되는 것 같은데 나중에 다시 보도록 하자.

- GET 방식 이외의 방식이 필터링이 안된다. + POST 방식 추가(외부 POST 걸러지는지는 확인 X)

 - $router = new Router(new Request);

    같은 방식을 통해서 HTTP 정보들을 부트스트래핑한 Request 객체를 사용하고 싶은데 잘 안되는 중.
    
    - 의존성 주입 이해 및 해결 (정적으로 생성했을시 의존성 주입, 동적으로 생성할 시 동적으로 오버로딩하여 사용)

- 스테틱으로 변수 저장이 안된다? ( DB쪽에서 설정파일을 통해 세팅할 때 안되는 중이다. )

- 라라벨에서는 Request가 어떤 방식으로 쓰이는지 확인할 것.

- 네임스페이스 사용할 것! 클래스별로 네임스페이스 추가해서 사용해볼 것.
